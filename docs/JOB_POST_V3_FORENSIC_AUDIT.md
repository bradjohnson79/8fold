## PHASE 5 — Root Cause

### Exact Root Cause: **G — Deployment / Code Version Mismatch**

**The fix commit (998fdc6) is not deployed to production.**

| Before 998fdc6 (deployed) | After 998fdc6 (local) | Production DB |
|---------------------------|------------------------|---------------|
| Table: `job_draft` | Table: `JobDraft` | Table: `JobDraft` |
| Columns: `user_id`, `created_at`, `updated_at` | Columns: `userId`, `createdAt`, `updatedAt` | Columns: `userId`, `createdAt`, `updatedAt` |

**What happens:**
1. Production runs code **without** 998fdc6 (pre-fix).
2. Drizzle generates SQL: `UPDATE job_draft SET ... WHERE user_id = $1` (snake_case).
3. Production DB has table `JobDraft` and columns `userId`, `createdAt`, `updatedAt`.
4. PostgreSQL error: `relation "job_draft" does not exist` (or `column "user_id" does not exist`).

**Ruled out:**
- A) Legacy table still active — Production has `JobDraft`, not `job_draft`.
- B) Route miswired — Route uses Drizzle `jobDraft` directly; no legacy repo.
- C) Enum mismatch — ACTIVE/ARCHIVED exist in production.
- D) Duplicate ACTIVE drafts — Possible but unrelated to 500.
- E) Constraint violation — Not the cause.
- F) Data corruption — Not the cause.
- **G) Migration/code not applied — Fix commit 998fdc6 not deployed.**
